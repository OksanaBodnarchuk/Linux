1.Ветка в Git это подвижный указатель на один из коммитов.
2.SSH-ключ нужен чтобы быстро устанавливать соединение с сервером (GitHub), не вводя пароля. Кроме того, SSH — единственный из сетевых протоколов, предоставляющий доступ и на чтение, и на запись.
3.GitHub Pages.позволяет публиковать живую демонстрацию кода в
виде сайта в Интернете.
Укажите команды, которые выполняют след. действия (GIT):
5.Создание, удаление веток:
Создать ветку: 
git branch <имя ветки>
Удаляет указанную ветку:
git branch -d <имя ветки>
6.Переключение между ветками:
git checkout <имя ветки>
7.Загрузка веток на GitHub:
git remote -v
git checkout <имя ветки>
git pull
git push
8.Удаление веток на GitHub:
git branch -D branch_name
Публикация сайта с помощью GitHub Pages:
1.Создание репозитория (либо навигация к существующему репозиторию)
2.Создание коммита маркдаун файла через веб интерфейс, как и с любым другим файлом
3.Активация GitHub Pages через настройки репозитория.
